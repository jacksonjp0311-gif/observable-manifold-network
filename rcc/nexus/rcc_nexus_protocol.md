# RCC Nexus Protocol

Before patching:

1. Identify task type.
2. Read README trisection.
3. Read repository context index.
4. Read Nexus index.
5. Read route map.
6. Read target folder README.
7. Inspect source/tests/evidence.
8. Patch smallest surface.
9. Run validation.
10. Update RCC-N records if geometry changed.

Boundary: route following is not validation.
